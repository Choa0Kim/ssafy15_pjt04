## 3. 구현 기능 설명

### [기본 기능: 자산별 커뮤니티]
*   **자산 목록 출력:** `assets.json`을 로드하여 주식, 채권, 원자재 등 다양한 자산군을 메인 화면에 표시합니다.
*   **실시간 카테고리 필터링:** Vanilla JS를 사용하여 페이지 새로고침 없이 자산군별로 모아볼 수 있는 탭 기능을 구현했습니다.
*   **자산별 게시판 CRUD:** 각 자산에 종속된 독립적인 게시판을 제공하며, 게시글의 작성/조회/수정/삭제가 가능합니다.
*   **커스텀 404 에러 페이지:** 존재하지 않는 자산이나 게시글 접근 시 금융기관 스타일의 정중한 에러 안내 페이지를 노출합니다.

### [심화 기능: LLM 콘텐츠 필터링]
*   **Upstage Solar API 연동:** 게시글 및 댓글 작성 시 LLM이 내용을 실시간 검사합니다.
*   **엄격한 비속어 차단:** 일반적인 욕설뿐만 아니라 'ㅗ', 'ㅅㅂ'와 같은 자음 욕설 및 변형된 비속어까지 감별하는 프롬프트 엔지니어링을 적용했습니다.

### [신규 기능: 스마트 머니 포트폴리오 대시보드]
*   **실제 데이터 파이프라인:** `yfinance`를 활용해 최신 시장 영업일을 반영하고, 워런 버핏, 국민연금(NPS) 등의 실제 공시 데이터를 기반으로 포트폴리오 정보를 구축했습니다.
*   **데이터 시각화:** `Chart.js`를 사용하여 자산 배분율을 직관적인 도넛 차트로 구현했습니다.
*   **포트폴리오별 토론 피드:** 각 투자 전략별로 실시간 댓글을 남기고 확인할 수 있는 비동기(AJAX/Fetch) 토론 기능을 추가했습니다.

## 4. 학습 내용
*   **Django CRUD Flow:** URL, View, Template, Model로 이어지는 Django의 핵심 흐름을 깊이 있게 이해했습니다.
*   **JSON & 외부 데이터 처리:** 외부 JSON 파일을 읽어와 DB 모델과 유연하게 연동하고, `yfinance` 라이브러리를 통해 실시간 금융 데이터를 가공하는 법을 익혔습니다.
*   **프롬프트 엔지니어링:** LLM의 페르소나를 설정하고 구체적인 검사 기준을 제시하여 한국어 비속어 필터링의 정확도를 높이는 경험을 했습니다.
*   **비동기 자바스크립트:** `fetch` API를 사용하여 사용자 경험(UX)을 해치지 않고 데이터를 갱신하는 현대적인 프론트엔드 통신 방식을 학습했습니다.

## 5. 트러블 슈팅
*   **문제:** `DEBUG = False` 설정 시 커스텀 404 페이지가 로드되지 않거나 정적 파일 설정 이슈 발생.
    *   **해결:** `ALLOWED_HOSTS`를 설정하고, `raise Http404`를 명시적으로 호출하여 Django의 에러 핸들링 메커니즘을 정확히 활용했습니다.
*   **문제:** LLM이 'ㅗ'와 같은 짧은 자음 비속어를 정상적인 문맥으로 오판하는 경우 발생.
    *   **해결:** `utils.py`의 프롬프트를 수정하여 '금융 커뮤니티의 품격'과 '자음 욕설'에 대한 명시적 기준을 부여하고, `temperature`를 0으로 설정하여 일관된 답변을 얻도록 조정했습니다.
*   **문제:** `replace` 도구 사용 시 중복된 문자열이나 퍼지 매칭으로 인한 오수정 발생.
    *   **해결:** `write_file`을 사용하여 파일 전체 내용을 명확하게 덮어쓰거나, 수정 전후의 컨텍스트를 충분히 제공하여 정확도를 높였습니다.
